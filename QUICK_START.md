# 🚀 Quick Start Guide - 5 Minutes to Deploy

Get your professional portfolio live in minutes!

## Step 1: Download Files (1 min)

You have these files ready:
- ✅ `index.html` - Main portfolio page
- ✅ `style.css` - Complete styling
- ✅ `script.js` - Interactive features
- ✅ `README.md` - Full documentation
- ✅ `.gitignore` - Git ignore file

## Step 2: Test Locally (1 min)

### On Your Computer
1. Keep all files in the same folder
2. Double-click `index.html` to open in your browser
3. **That's it!** Portfolio should load immediately

### Optional: Use Live Server
If you have VS Code:
1. Open the folder in VS Code
2. Right-click `index.html`
3. Select "Open with Live Server"

## Step 4: Deploy to GitHub Pages (1 min)

### Option A: GitHub Web Interface (Easiest)

1. **Create GitHub Account** (if you don't have one)
   - Go to github.com
   - Sign up (free)

2. **Create New Repository**
   - Click "+" in top right
   - Select "New repository"
   - Name: `portfolio` (or `username.github.io` for custom URL)
   - Check "Add README file"
   - Click "Create repository"

3. **Upload Files**
   - Click "Add file" → "Upload files"
   - Drag & drop all files (index.html, style.css, script.js)
   - Scroll down, click "Commit changes"

4. **Enable GitHub Pages**
   - Go to repository "Settings"
   - Left menu: "Pages"
   - Source: Select "main" branch
   - Click "Save"
   - Wait 1-2 minutes for deployment

5. **Get Your URL**
   - Your portfolio is live at:
   - `https://username.github.io/portfolio`
   - (or `https://username.github.io` if using username.github.io repo)

### Option B: Command Line (For Developers)

```bash
# Navigate to your portfolio folder
cd path/to/portfolio

# Initialize git
git init

# Add all files
git add .

# Create first commit
git commit -m "Deploy portfolio website"

# Add remote (replace with your repo URL)
git remote add origin https://github.com/username/portfolio.git

# Push to GitHub
git branch -M main
git push -u origin main
```

Then follow the GitHub Pages setup steps above (step 4).

## 🆘 Quick Troubleshooting

### Portfolio won't open locally
- ❌ Make sure all files are in the same folder
- ✅ Try double-clicking `index.html` directly

### GitHub Pages not showing
- ❌ Check if Settings → Pages is enabled
- ❌ Verify you're looking at the correct URL
- ✅ Wait 2-3 minutes for deployment
- ✅ Hard refresh: Ctrl+Shift+R (Windows) or Cmd+Shift+R (Mac)

### Styles or images not showing on GitHub
- ❌ Check file paths are relative (not absolute)
- ❌ Verify file names match exactly
- ✅ Commit files again and wait for redeployment

### Contact form not working
- ❌ Update the email address in the form
- ✅ Form uses mailto links (opens email client)
- ✅ Test with your actual email address
